## I also recommend one architecture improvement

Right now, the Add screen's recent transactions are built by dynamically creating `Button` widgets in Python (`refresh_add_screen_history()`), while the Dashboard uses a reusable `DashboardTransactionRow` component.  

I recommend creating a reusable `AddRecentTransactionRow` widget (similar to `DashboardTransactionRow`) and populating it instead of creating `Button` objects manually. This keeps the UI consistent and makes future features like long-press, swipe, or context menus much easier to add.

This will also eliminate duplicated UI code and move your project closer to a clean, component-based design.
